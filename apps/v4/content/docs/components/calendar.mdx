## التاريخ المحدد (مع المنطقة الزمنية)

يقبل مكون التقويم خاصية `timeZone` لضمان عرض التواريخ واختيارها في المنطقة الزمنية المحلية للمستخدم.

```tsx showLineNumbers
export function CalendarWithTimezone() {
  const [date, setDate] = React.useState<Date | undefined>(undefined)
  const [timeZone, setTimeZone] = React.useState<string | undefined>(undefined)

  React.useEffect(() => {
    setTimeZone(Intl.DateTimeFormat().resolvedOptions().timeZone)
  }, [])

  return (
    <Calendar
      mode="single"
      selected={date}
      onSelect={setDate}
      timeZone={timeZone}
    />
  )
}
```

**ملاحظة:** إذا لاحظت إزاحة في التاريخ المحدد (على سبيل المثال، تحديد اليوم العشرين يبرز اليوم التاسع عشر)، فتأكد من تعيين خاصية `timeZone` على المنطقة الزمنية المحلية للمستخدم.

**لماذا من جانب العميل؟** يتم الكشف عن المنطقة الزمنية باستخدام `Intl.DateTimeFormat().resolvedOptions().timeZone` داخل `useEffect` لضمان التوافق مع العرض من جانب الخادم. سيؤدي الكشف عن المنطقة الزمنية أثناء العرض إلى عدم تطابق الترطيب، حيث قد يكون الخادم والعميل في مناطق زمنية مختلفة.

## دليل الترقية

### Tailwind v4

إذا كنت تستخدم Tailwind v4 بالفعل، فيمكنك الترقية إلى أحدث إصدار من مكون `Calendar` عن طريق تشغيل الأمر التالي:

```bash
npx shadcn@latest add calendar
```

عندما يُطلب منك الكتابة فوق مكون `Calendar` الموجود، حدد `Yes`. **إذا قمت بإجراء أي تغييرات على مكون `Calendar`، فستحتاج إلى دمج تغييراتك مع الإصدار الجديد.**

سيؤدي هذا إلى تحديث مكون `Calendar` و `react-day-picker` إلى أحدث إصدار.

بعد ذلك، اتبع دليل ترقية [React DayPicker](https://daypicker.dev/upgrading) لترقية مكوناتك الحالية إلى أحدث إصدار.

#### تثبيت الكتل

```bash
npx shadcn@latest add calendar-02
```

سيؤدي هذا إلى تثبيت أحدث إصدار من كتل التقويم.

```tsx showLineNumbers title="components/ui/calendar.tsx"
"use client"

import * as React from "react"
import {
  ChevronDownIcon,
  ChevronLeftIcon,
  ChevronRightIcon,
} from "lucide-react"
import { DayButton, DayPicker, getDefaultClassNames } from "react-day-picker"

import { cn } from "@/lib/utils"
import { Button, buttonVariants } from "@/components/ui/button"

function Calendar({
  className,
  classNames,
  showOutsideDays = true,
  captionLayout = "label",
  buttonVariant = "ghost",
  formatters,
  components,
  ...props

  buttonVariant?: React.ComponentProps<typeof Button>["variant"]
}) {
  const defaultClassNames = getDefaultClassNames()

    <DayPicker
      showOutsideDays={showOutsideDays}

          "relative flex flex-col gap-4 md:flex-row",

function CalendarDayButton({
  className,
  day,
  modifiers,
  ...props
}: React.ComponentProps<typeof DayButton>) {
  const defaultClassNames = getDefaultClassNames()

  const ref = React.useRef<HTMLButtonElement>(null)
  React.useEffect(() => {
    if (modifiers.focused) ref.current?.focus()
  }, [modifiers.focused])

  return (
    <Button
      ref={ref}
      variant="ghost"
      size="icon"
      data-day={day.date.toLocaleDateString()}
      data-selected-single={
        modifiers.selected &&
        !modifiers.range_start &&
        !modifiers.range_end &&
        !modifiers.range_middle
      }
      data-range-start={modifiers.range_start}
      data-range-end={modifiers.range_end}
      data-range-middle={modifiers.range_middle}
      className={cn(
        "data-[selected-single=true]:bg-primary data-[selected-single=true]:text-primary-foreground data-[range-middle=true]:bg-accent data-[range-middle=true]:text-accent-foreground data-[range-start=true]:bg-primary data-[range-start=true]:text-primary-foreground data-[range-end=true]:bg-primary data-[range-end=true]:text-primary-foreground group-data-[focused=true]/day:border-ring group-data-[focused=true]/day:ring-ring/50 flex aspect-square h-auto w-full min-w-[--cell-size] flex-col gap-1 font-normal leading-none data-[range-end=true]:rounded-md data-[range-middle=true]:rounded-none data-[range-start=true]:rounded-md group-data-[focused=true]/day:relative group-data-[focused=true]/day:z-10 group-data-[focused=true]/day:ring-[3px] [&>span]:text-xs [&>span]:opacity-70",
        defaultClassNames.day,
        className
      )}
      {...props}
    />
  )
}

export { Calendar, CalendarDayButton }
```

### 2025-10-26 إصلاح نصف قطر اليوم مع أرقام الأسابيع

لقد أصلحنا مشكلة حيث لم يتم تطبيق نصف قطر الحد الأيسر لليوم المحدد بشكل صحيح عند عرض أرقام الأسابيع. يضمن الإصلاح أنه عند تمكين `showWeekNumber`، فإن اليوم الأول (وهو الطفل الثاني بسبب عمود رقم الأسبوع) يتلقى بشكل صحيح حدًا أيسر مستديرًا.

لتطبيق هذا الإصلاح، قم بتحرير `components/ui/calendar.tsx` وقم بتحديث فئة `day` في `classNames`:
