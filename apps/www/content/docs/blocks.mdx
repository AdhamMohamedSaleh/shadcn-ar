ندعو المجتمع للمساهمة في [مكتبة البلوكات](/blocks). شارك مكوناتك وبلوكاتك مع مطورين آخرين وساعد في بناء مكتبة مكونات عالية الجودة وقابلة لإعادة الاستخدام.

نرحب بجميع أنواع البلوكات: تطبيقات، تسويق، منتجات، وأكثر.

## إعداد بيئة العمل

<Steps>

### نسخ المستودع

```bash
git clone https://github.com/shadcn-ui/ui.git
```

### أنشئ فرعًا جديدًا

```bash
git checkout -b username/my-new-block
```

### تثبيت التبعيات

```bash
pnpm install
```

### تشغيل الخادم التطويري

```bash
pnpm www:dev
```

</Steps>

## إضافة بلوك

يمكن أن يكون البلوك عبارة عن مكون بسيط (مثال: نسخة معدّلة من مكون واجهة) أو مكون مركب مع عدة عناصر، مثل لوحة تحكم مع مكونات وخطافات وأدوات مساعدة.

<Steps>

### أنشئ مجلدًا جديدًا للبلوك

قم بإنشاء مجلد جديد في المسار `apps/www/registry/new-york/blocks`. يجب أن يكون اسم المجلد بصيغة kebab-case وتحت مجلد `new-york`.

```txt
apps
└── www
    └── registry
        └── new-york
            └── blocks
                └── dashboard-01
```

<Callout className="mt-6">

**ملاحظة:** برنامج build سيتولى بناء البلوك للنمط الافتراضي.

</Callout>

### أضف ملفاتك

أضف ملفاتك إلى مجلد البلوك. هنا مثال على بلوك به صفحة ومكونات وخطافات ودوال مساعدة.

```txt
dashboard-01
└── page.tsx
└── components
    └── hello-world.tsx
    └── example-card.tsx
└── hooks
    └── use-hello-world.ts
└── lib
    └── format-date.ts
```

<Callout className="mt-6">

**ملاحظة:** يمكنك البدء بملف واحد ثم إضافة المزيد لاحقًا.

</Callout>

</Steps>

## أضف البلوك إلى السجل

<Steps>

### أضف تعريف البلوك إلى `registry-blocks.tsx`

لإضافة البلوك إلى السجل يجب إضافة تعريفه لملف `registry-blocks.ts`.

يعتمد ذلك على مخطط السجل المشروح هنا: [https://ui.shadcn.com/schema/registry-item.json](https://ui.shadcn.com/schema/registry-item.json).

```tsx title="apps/www/registry/registry-blocks.tsx"
export const blocks = [
  // ...
  {
    name: "dashboard-01",
    author: "shadcn (https://ui.shadcn.com)",
    title: "لوحة تحكم",
    description: "لوحة تحكم بسيطة مع مكون hello world.",
    type: "registry:block",
    registryDependencies: ["input", "button", "card"],
    dependencies: ["zod"],
    files: [
      {
        path: "blocks/dashboard-01/page.tsx",
        type: "registry:page",
        target: "app/dashboard/page.tsx",
      },
      {
        path: "blocks/dashboard-01/components/hello-world.tsx",
        type: "registry:component",
      },
      {
        path: "blocks/dashboard-01/components/example-card.tsx",
        type: "registry:component",
      },
      {
        path: "blocks/dashboard-01/hooks/use-hello-world.ts",
        type: "registry:hook",
      },
      {
        path: "blocks/dashboard-01/lib/format-date.ts",
        type: "registry:lib",
      },
    ],
    categories: ["dashboard"],
  },
]
```

تأكد من إضافة اسم، وصف، نوع، تبعيات السجل، تبعيات، ملفات، وتصنيفات. سنناقش كل خاصية في وثائق المخطط (قريبًا).

### شغل سكربت البناء

```bash
pnpm registry:build
```

<Callout className="mt-6">

**ملاحظة:** لست بحاجة لتشغيل هذا السكربت في كل مرة تعدّل فيها الملفات، فقط عند تحديث تعريف البلوك.

</Callout>

### استعرض البلوك الخاص بك

بعد انتهاء السكربت يمكنك استعراض البلوك على `http://localhost:3333/blocks/[CATEGORY]` أو عرض كامل الصفحة من خلال `http://localhost:3333/view/styles/new-york/dashboard-01`.

### بناء البلوك

يمكنك الآن بناء البلوك من خلال تعديل الملفات الموجودة في مجلد البلوك ومتابعة التغيرات في المتصفح.

إذا أضفت ملفات جديدة تأكد من إضافتها إلى مصفوفة `files` في تعريف البلوك.

</Steps>

## نشر البلوك

عند الانتهاء من تطوير البلوك يمكنك إرسال Pull Request إلى المستودع الرئيسي.

<Steps>

### شغل سكربت البناء

```bash
pnpm registry:build
```

### التقط صورة للبلوك

```bash
pnpm registry:capture
```

<Callout className="mt-6">

**ملاحظة:** إذا شغلت السكربت من قبل قد تحتاج لحذف الصور القديمة من `apps/www/public/r/styles/new-york` وتشغيل السكربت مرة أخرى.

</Callout>

### إرسال Pull Request

قم برفع تغييراتك وأرسل Pull Request إلى المستودع الرئيسي.

سيتم مراجعة دمج البلوك وبعدها يظهر في الموقع ويمكن تثبيته عبر أداة CLI.

</Steps>

## التصنيفات

خاصية `categories` تُستخدم لتنظيم البلوك داخل السجل.

### إضافة تصنيف

لإضافة تصنيف جديد أضفه إلى مصفوفة `registryCategories` في `apps/www/registry/registry-categories.ts`.

```tsx title="apps/www/registry/registry-categories.ts"
export const registryCategories = [
  // ...
  {
    name: "Input",
    slug: "input",
    hidden: false,
  },
]
```

## إرشادات

فيما يلي بعض الإرشادات عند المساهمة في مكتبة البلوكات:

- الخصائص المطلوبة لتعريف البلوك: `name`، `description`، `type`، `files`، و `categories`.
- تأكد من إدراج جميع تبعيات السجل ضمن `registryDependencies`، مثل اسم المكون في السجل (مثال: input، button، card).
- تأكد من ذكر جميع التبعيات في `dependencies` (اسم الحزمة مثل zod، sonner...).
- إذا كان للبلوك صفحة (اختياري)، يجب أن تكون أول عنصر في مصفوفة `files` ولها خاصية `target` لمساعدة أداة CLI على وضعها في المكان الصحيح.
- **يجب أن تستخدم الاستيرادات دومًا مسار `@/registry`.** مثال: `import { Input } from "@/registry/new-york/input"`
