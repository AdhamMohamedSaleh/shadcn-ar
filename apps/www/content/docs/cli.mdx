## init

استخدم أمر `init` لتهيئة الإعدادات والتبعيات لمشروع جديد.

يقوم أمر `init` بتثبيت التبعيات، إضافة أداة `cn`، وضبط متغيرات CSS للمشروع.

```bash
npx shadcn@latest init
```

### الخيارات

```txt
استخدام: shadcn init [الخيارات] [المكونات...]

تهيئة مشروعك وتثبيت التبعيات

المعطيات:
  components         المكونات التي تود إضافتها أو رابط المكون

الخيارات:
  -y, --yes           تجاوز رسالة التأكيد. (الافتراضي: true)
  -d, --defaults      استخدام الإعدادات الافتراضية. (الافتراضي: false)
  -f, --force         إجبار تحديث الإعدادات الموجودة. (الافتراضي: false)
  -c, --cwd <cwd>     مسار العمل الحالي. الافتراضي هو المسار الحالي. (الافتراضي: "/Users/shadcn/Desktop")
  -s, --silent        كتم الناتج. (الافتراضي: false)
  --src-dir           استخدم مجلد src. (الافتراضي: false)
  --no-src-dir        لا تستخدم مجلد src.
  --css-variables     استخدم متغيرات css للثيمات. (الافتراضي: true)
  --no-css-variables  لا تستخدم متغيرات css للثيمات.
  -h, --help          عرض المساعدة للأمر
```

## add

استخدم أمر `add` لإضافة مكونات وتبعيات إلى مشروعك.

```bash
npx shadcn@latest add [component]
```

### الخيارات

```txt
استخدام: shadcn add [الخيارات] [المكونات...]

إضافة مكون لمشروعك

المعطيات:
  components         المكونات المراد إضافتها أو رابط المكون

الخيارات:
  -y, --yes           تجاوز رسالة التأكيد. (الافتراضي: false)
  -o, --overwrite     استبدال الملفات الموجودة. (الافتراضي: false)
  -c, --cwd <cwd>     مسار العمل الحالي. الافتراضي هو المسار الحالي. (الافتراضي: "/Users/shadcn/Desktop")
  -a, --all           إضافة جميع المكونات المتاحة (الافتراضي: false)
  -p, --path <path>   المسار لإضافة المكون إليه.
  -s, --silent        كتم الناتج. (الافتراضي: false)
  --src-dir           استخدم مجلد src. (الافتراضي: false)
  --no-src-dir        لا تستخدم مجلد src.
  --css-variables     استخدم متغيرات css للثيمات. (الافتراضي: true)
  --no-css-variables  لا تستخدم متغيرات css للثيمات.
  -h, --help          عرض المساعدة للأمر
```

## build

استخدم أمر `build` لإنشاء ملفات JSON الخاصة بالسجل.

```bash
npx shadcn@latest build
```

يقرأ هذا الأمر ملف `registry.json` وينشئ ملفات JSON الخاصة بالسجل في مجلد `public/r`.

### الخيارات

```txt
استخدام: shadcn build [الخيارات] [registry]

بناء المكونات لسجل shadcn

المعطيات:
  registry             مسار ملف registry.json (الافتراضي: "./registry.json")

الخيارات:
  -o, --output <path>  مسار مجلد الإخراج لملفات json (الافتراضي: "./public/r")
  -c, --cwd <cwd>      مسار العمل الحالي (الافتراضي: "/Users/shadcn/Code/shadcn/ui/packages/shadcn")
  -h, --help           عرض المساعدة للأمر
```

لتخصيص مجلد الإخراج، استخدم خيار `--output`.

```bash
npx shadcn@latest build --output ./public/registry
```
